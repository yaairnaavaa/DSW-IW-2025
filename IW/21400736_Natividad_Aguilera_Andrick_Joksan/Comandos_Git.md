# comandos git

1. # git init
Inicializa un nuevo repositorio Git en el directorio actual.
# Ejemplo:
git init en una nueva carpeta para comenzar un proyecto con control de versiones.


2. # git clone
Clona un repositorio remoto a tu máquina local.
# Ejemplo:
git clone https://github.com/usuario/repositorio.git


3. # git status
Muestra el estado actual del repositorio y los archivos modificados.
# Ejemplo:
git status para ver qué archivos han sido modificados o agregados.


4. # git add
Agrega archivos al área de preparación (staging area).
# Ejemplo:
git add archivo.txt para incluir el archivo en el próximo commit.


5. # git commit
Guarda los cambios preparados en el historial del repositorio.
# Ejemplo:
git commit -m "Agrega nueva funcionalidad"


6. # git push
Envía los commits locales al repositorio remoto.
# Ejemplo:
git push origin main para subir los cambios a la rama principal.


7. # git pull
Descarga e integra cambios desde el repositorio remoto.
# Ejemplo:
git pull origin main para traer los últimos cambios.


8. # git fetch
Descarga los cambios del repositorio remoto sin fusionarlos.
# Ejemplo:
git fetch origin para obtener actualizaciones del remoto.


9. # git merge
Fusiona una rama con la rama actual.
# Ejemplo:
git merge rama-secundaria para integrar sus cambios.


10. # git branch
Muestra, crea o elimina ramas.
# Ejemplo:
git branch nueva-rama para crear una nueva rama.


11. # git checkout
Permite cambiar de rama o restaurar archivos.
# Ejemplo:
git checkout develop para cambiar a la rama “develop”.


12. # git log
Muestra el historial de commits.
# Ejemplo:
git log para ver los commits realizados en el repositorio.


13. # git diff
Muestra las diferencias entre versiones de archivos.
# Ejemplo:
git diff archivo.txt para ver los cambios sin confirmar.


14. # git reset
Revierte el área de preparación o elimina commits.
# Ejemplo:
git reset --hard HEAD~1 para eliminar el último commit.


15. # git revert
Crea un nuevo commit que deshace los cambios de un commit anterior.
# Ejemplo:
git revert abc1234 para revertir un commit específico.


16. # git remote
Gestiona los repositorios remotos.
# Ejemplo:
git remote add origin https://github.com/usuario/repo.git


17. # git stash
Guarda cambios temporales sin hacer commit.
# Ejemplo:
git stash para guardar tus cambios antes de cambiar de rama.


18. # git tag
Crea marcas en commits importantes, como versiones.
# Ejemplo:
git tag v1.0 para marcar la versión 1.0 del proyecto.


19. # git config
Configura opciones de Git como nombre de usuario o correo.
# Ejemplo:
git config --global user.email "ejemplo@correo.com"


20. # git rm
Elimina archivos del repositorio.
# Ejemplo:
git rm archivo.txt para eliminarlo y registrar el cambio.
